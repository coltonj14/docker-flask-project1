Testing with Pytest

Testing with pylint and furthermore testing in general is very important for a project. These tests actually assert that what you have coded in on the page when it is loaded. In combination with continuous integration and deployment testing is a great way to catch bugs and just make sure everything you need is on the page and working correctly. In the code I am using pytest to create assertions for element on my page. Whether this be items in the nav bar or actual words, buttons, or content on the page itself. Below is an example of what one of these assertions looks like:

response = client.get("/")
 assert response.status_code == 200
 assert b'<a class="nav-link" href="/about">About' in response.data

Project Files

docker_flask_project1 <- root of the project

  • .github directory <- directory to work with github actions
    • workflows <- directory that houses the CI/CD files
      • dev.yml <- activates on pull request and runs pytests as well as deploys to heroku development app
      • prod.yml <- activates on a marge or push to main and runs pytests, deploys to heroku production app,
  • app directory <- houses the main files of the application
    • context_processors <- Gets values that will be used in the rest of the project
      • __init__.py <- gets information that can be used in the rest of the application by reference
    • simple pages <- the directory of the pages of the application
      • about.html <- html for the about page
      • index.html <- html for the home page
      • page1.html <- html for page 1 - git
      • page2.html <- html for page 2 - docker
      • page3.html <-html for page 3 - python/flask
      • page4.html <-html for page 4 - CI/CD
      • __init__.py <- starting of flask
    • static
      • css <- houses the css file for project
        • style.css <- file for webpage styling
      • images <- directory of the images used in the project
      • js <- houses javascript files
    • templates
      • base.html <- provides the base layout for all of the pages
    • __init__.py <- starting flask instance
    • run.py <- creates the app
  • tests directory
    • conftest.py <- tests the configuration of the app
    • context_process_test.py <- tests the context process values that we calculated
    • simple_pages_test.py <- test and creates assertions for each of the webpages
  • rest of files
    • .coveragerc <- control coverage.py
    • .dockerignore <- files docker should ignore
    • .gitignore <- files git should ignore
    • .pylintrc <- outline pylint rules
    • docker-compose.yml <- Orchestration file that contains the configuration to develop locally, it overrides the Dockerfile to run the flask development server instead of running the gunicorn server that is used for hosting on Heroku
    • Dockerfile <- File used to create an image to run in a container that runs the program
    • heroku.yml <- gives necessary commands/information to heroku in order to run application
    • pytest.ini <- pytest configuration
    • readme.md <- Instructions for project
    • requirements.txt <- list of dependencies for the project need to install these before running
