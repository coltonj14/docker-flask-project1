CI/CD

Continous Integration and Deployment

For this project I have set up Github actions which allows me to run tests when merging and when creating pull requests. For this I have set up my workflow files (dev.yml and prod.yml) with my information from dockerhub, heroku, and Github. Through these .yml files and Github actions I am able to perform tests and push to my development heroku application all before I merge my code with main. Once I confirm the merge I have more Github actions that test on the merge and deploy to my production heroku application.

Push to Docker

In order to push to my docker repository I have in my prod.yml file an action to log into docker and push to the specific image that I have on dockerhub for my repository.
