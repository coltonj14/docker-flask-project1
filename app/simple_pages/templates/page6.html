AAA Testing

AAA and Why Test?

AAA testing is a process or framework for testing code. It represents three parts A(Arrange) A(Act) and A(Assert). These are the three parts that can ensure a good test. AAA is powerful because it teaches the importance of a simple yet clear test. It also helps to separate your code into tests that test one thing.Testing code is important in order to make sure that everything you are writing, code wise, can be relied upon and is robust. Testing can also help to find where your code is failing or wether it is producing the expected output or not.

Arrange

The first A is for Arranging that data in order to be tested. This should essentially set up the test case. This section should handle everything from special setup to potentially pulling from a database. An example of this from the calculator project would be setting up the numbers that we want to perform operations on. This could be setting up the data structure or evem just making sure you have the correct numbers to test with.

Act

Now it is time to Act. This portion is now acting on the data that we have set up in the arrange portion of the testing framework. This could be doing something like calling a function or performing an operation on the data. This step in our framework will supply our assertion.

Assert

Assertions are essentially testing whether the result from the act step will pass or fail. The act step, as mentioned, should produce a response that can be asserted. This assertion will then indicate whether the test has passed or failed. An example from the calculator is testign whether an operation on two numbers produces the expected result.

Concise vs Verbose

When writing testing in the AAA style/format, it may be easier to understand if this process is coded verbosely. This thought process can bring up issues though as it increases code size. When the code is written more concisely it helps to keep your functions or methods doing only one thing. This in the end, makes it easier to test.

AAA in Python

With this specific calculator project, we have incorporated AAA testing. The arrange and act phase can be seen through the code that preps the data for testing and provides the action. We call the create() method in the Calculations class to set up the data so that get_result() can access the instance variable that is created to test it. This is the implementation in the classes itself that makes it easier to test. As for the actual test itself we create the tuple list and then call the create method on a particular operation. We then assert on this action.
