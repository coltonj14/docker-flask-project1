Docker

Docker

Docker provides applications with a container to package up and run in. This is an isolated environment that is called a container. This is great as everything you need is in the container regardless of what computer you are using.

Images

An image is an isolated filesystem that runs in the container. It also houses everything needed to run the container including environment variables and dependencies.

Docker commands

  • docker run - This command allows you to start up a container using a specified image. It usually will come in the form docker run -dp port:port {image-name}
  • docker build - This command will build your container based on some variable sin your Dockerfile. your Dockerfile can contain some setup variables that will help to install dependencies that are needed.
  • docker ps - docker ps will allows you to see the running docker containers. From this command you can also get information about your running containers. This information can include container IDs.
  • docker stop - stop a container from running
  • docker rm {container id} - This command allows you to remove a particular container. You can also do this from the docker desktop application
  • docker push - push your container to dockerhub using this command in pair with your repository name
  • docker tag - This command allows you to add a tag to your image to give it a new name
  • docker network - allows container to be on the same "network" which in turn allows them to communicate with eachother
  • docker exec -it - allows you to access a terminal inside of the container

Starting the Project

In order to start the project in your IDE using the command line you will use the command docker compose up --build In order to access the container from the terminal you can use the command docker ps in order to get the container name. Then use the command docker exec -it {container name} /bin/bash to access a terminal in the container.
