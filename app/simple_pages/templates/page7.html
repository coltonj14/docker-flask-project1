Calculator OOP principles

In this section I will go over how the calculator portion of this project demonstrates the principles of Object Oriented Design.

Encapsulation

With encapsulation we are focusing on the hiding of data by encapsulating it in the class. In the Calculator implementation this can easily be seen in the Calculations class. We use the method def get_result(self) in order to access the instance variable list to get the result of the calculation. This aspect of the project also demonstrates another priciple of OOP, polymorphism.

Polymorphism

As stated about this example also demonstrates polymorphism. Each individual Calculation subclass has its own def get_result(self) method. For an illustration look at the code for the addition and subtraction.

  • Addition -
    sum_of_values = 0.0
    for value in self.values:
      sum_of_values = Add.add(value, sum_of_values)
    return sum_of_values
  • Subtraction -
    difference_of_values = 0.0
    for value in self.values:
      difference_of_values = Sub.subtract(difference_of_values, value)
    return difference_of_values
As we can see the implementation is a little bit different but the method signature is the same for both of these methods.

Abstraction

The Calculation class is an abstract class that provides the blueprint for each of it's subclasses. In this case it provides the constructor and create methods for other classes to use to create the instance variable values.

Inheritance

Each of the classes Addition, Subtraction, Multiplication, and Division in this file are subclasses on Calculation this can be seen in the method declaration: class Addition(Calculation): This means that each one of these subclasses inherits data and methods from the parent class Calculation. This is clear because each of these subclasses has access to the instance variable values
